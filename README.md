# Car OBD Diagnostics


## Overview

This project is a Unity-based car diagnostics dashboard prototype. It simulates raw OBD telemetry, decodes the incoming values into structured automotive parameters, and displays the results in a UI Toolkit dashboard. A local LLM connection through LM Studio is used to generate human-readable diagnostic summaries from the current vehicle state.

The project is currently designed as a demo and development environment using virtual telemetry values. It can be extended later to support real OBD hardware input.

## Features

- Simulated OBD telemetry generation
- Raw OBD frame encoding and decoding
- Decoded vehicle parameter dashboard
- Local diagnostics summary generation through LM Studio
- Scenario simulation for common vehicle issues
- Activity log for recent telemetry events
- Unity UI Toolkit-based interface

## Project Flow

1. Virtual OBD data is generated by the simulator.
2. The simulator encodes the snapshot into a raw OBD-style frame.
3. The decoder converts the raw frame back into readable automotive telemetry.
4. The dashboard displays the decoded values and derived status summary.
5. A diagnostics prompt can be sent to LM Studio for a natural-language summary.

## Main Scripts

### `Assets/Scripts/OBDSimulator.cs`

Generates virtual vehicle telemetry and injects scenario-specific fault patterns.

### `Assets/Scripts/OBDDecoder.cs`

Encodes structured values into a raw frame and decodes raw values back into readable telemetry.

### `Assets/Scripts/OBDParameters.cs`

Defines the main decoded vehicle diagnostics data model.

### `Assets/Scripts/DashboardManager.cs`

Controls the dashboard UI, tab navigation, prompt generation, scenario switching, and activity logging.

### `Assets/Scripts/LMStudioClient.cs`

Sends diagnostic prompts to a local LM Studio endpoint and returns the model response.

## Dashboard Tabs

### Diagnostics

Shows:

- decoded summary

- local diagnostics summary
- prompt input and send controls

### Scenarios

Lets the user switch between simulated vehicle fault conditions.

### Activity Log

Displays recent runtime telemetry activity.

## Local Model Setup

This project expects LM Studio to be running locally with an OpenAI-compatible endpoint.

Default endpoint used by the project:

`http://localhost:1234/v1/chat/completions`

If the local model service is unavailable, the dashboard will still run, but the natural-language diagnostics summary will not return a response.

## Current Limitations

- Uses simulated OBD data instead of live hardware input
- No persistent logging or export pipeline
- No formal automated tests yet
- Local model output is unstructured text
